04 · The framework

A structured conversation,
in four stages.

Designed to make the interface type decision explicit, documented, and impossible to override by opinion alone.

Principle
Values come first.
Before any technical question.
Based on Ackoff (1989) · DIKW
01 —
Values
Rank what the organisation stands for. That ranking is the primary filter.
02 —
Context
Eleven structured questions: users, frequency, environment, consequence of failure.
03 —
Patterns
Does this context resemble any of the five documented failures? The framework flags the match.
04 —
Report
Not a verdict. Alignments, tensions, precedents. The designer reads it and decides.
NEXUS · Defense06
05 · Research method

Three phases.
Three kinds of evidence.

PHASE 01 Construction
Built from the literature
Literature review confirms no established framework addresses interface type selection as a standalone decision. Expert interviews with practitioners produce the values taxonomy and assessment structure.
15 designers & PMs interviewed
PHASE 02 Retrospective
Tested against history
Framework applied to the five documented failures using only information publicly available before each product launched. Target: identify 70 % of documented failure factors (Landis & Koch, 1977).
5 cases · pre-launch data only
PHASE 03 Proof of concept
Validated with practitioners
NEXUS built as a browser-based tool and tested with designers and product managers. Works in practice, not only on paper — and produced the usability data on the next slide.
15 participants · tool session

08 · Contributions

Four contributions.
One open platform.

For the field
A replicable method
First application of DIKW methodology (Ackoff, 1989) to interface type selection. Extensible to new types, cultural contexts, and failures we have not yet seen.
For practice
Evidence to defend good decisions
A structured argument for designers. Makes judgment visible, documented, and harder to override by opinion alone — including the HiPPO's.
For the community
Infrastructure that outlives the defense
MIT-licensed tool accessible to a solo designer and a corporation equally. The research does not end when this session does.
As a position
"It informs. It does not decide."
Structure in service of human judgment, not instead of it.
